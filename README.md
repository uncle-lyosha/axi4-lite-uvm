# AXI4-LITE
## RTL
Стандартный протокол AXI4-lite (описание данного протокола лежит в папке doc), выполненный в два FSM блока (один для чтения, другой для записи) и memory map. Каждый FSM написан в три блока always: первый отвечает за сброс состояния режима; второй реализован в комбинационной логике, защищая от latch, и отвечает за переключение состояний; третий с параллельной логикой отвечает за состояния сигналов. Подробнее почитать о данной реализации FSM можно в статьях Cliford E. Cummings.
### Memory map:		
|	ADDR	| 		REG		  |		Function		|
|	:---	| 	:---		  |	  	:---  		|
|	0x00	| 	CTRL		  |			W/R				|	
|	0x04	| 	STATUS	  |			 R				|		
|	0x08	|  	DATA_IN	  |			W/R				|
|	0x0C	| 	DATA_OUT 	|			 R				|

## UVM
Архитектура UVM стандартная состоит из двух агентов для шины чтения и шины записи, сборщиков покрытия для чтения и записи, один scoreboard и RAL без адаптера. Верификационный план представлен в папке doc.
### RAL
Чтобы обращаться к регистру STATUS и DATA_OUT добавлена RAL модель (only backdoor). От адаптера и реализации frontdoor было принято отказаться в связи с ненужностью в этой системе, так как транзакции постоянно записывают значения в регистры CTRL и DATA_IN. RAL модель использовалась в мониторе (для считывания данных с нужного регистра и отправки этих данных на проверку в scoreboard) и в sequnce для записи в регистры STATUS и DATA_OUT (чтобы проверить, правильно ли считываются данные с них интерфейсом AXI).

## Запуск
### Основные команды
При запуске тестирования папка sim с данными о симуляции создастся автоматически.
```bash
  # Запуск регрессии (всех тестов) в CLI режиме
  make regress
```

```bash
  # Запуск в CLI режиме основного теста
  make run
```

```bash
  # Запуск в GUI режиме для отладки
  make gui
```

```bash
  # Очистка папки sim
  make clean
```
### Дополнительные команды
```bash
  # Запуск тестбеча с нужным нам тестом (использвать с make run и make gui)
  TEST=test_name
```
```bash
  # Запуск тестбенча с заданным сидом (по умолчанию сид случайны)
  SEED=123456
```
